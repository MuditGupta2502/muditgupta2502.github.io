Types of Databases, Abstraction, Interfaces, and Users

Foundations | 18 min read

Why This Article Exists

This article consolidates missing foundation concepts: types of databases, data abstraction layers, DBMS interfaces, and database user roles. These are mandatory before advanced design and optimization.

Types of Databases

  • Centralized databases: one server, simple management, potential single point of failure.
  • Distributed databases: data spread across nodes for scale and fault tolerance.
  • Relational databases: schema-driven tables with SQL and ACID transactions.
  • NoSQL databases: flexible schemas for high-volume or semi-structured workloads.
  • Operational vs analytical stores: OLTP for transactions, OLAP for reporting.

Data Abstraction

Three-level abstraction separates how users view data from how data is physically stored:

  1. External level: user-specific view.
  2. Conceptual level: global logical schema.
  3. Internal level: storage/index organization.

DBMS Interfaces

  • CLI SQL shells for power users.
  • GUI admin consoles for schema and privilege tasks.
  • Programmatic interfaces: JDBC, ODBC, ORM layers.
  • APIs and microservices exposing domain-safe operations.

What to Remember

Strong systems start with clear database type selection, layered abstraction, and clean role boundaries.
