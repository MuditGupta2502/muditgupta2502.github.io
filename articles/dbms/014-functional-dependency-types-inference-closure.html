Functional Dependency: Types, Inference Rules & Attribute Closure

Functional dependencies (FDs) are the formal heart of normalization theory. What is a Functional Dependency?

A functional dependency X â†’ Y holds in a relation R if, for every valid extension (set of rows) of R, whenever two rows agree on X, they must also agree on Y.

Important: FDs are semantic constraints â€” they represent business rules, not just what happens to be true in the current data. If today one student happens to have one advisor, that doesn't mean student_id â†’ advisor is an FD unless it's a business rule that every student has exactly one primary advisor.

Types of Functional Dependencies

1. Trivial FD

An FD X â†’ Y is trivial if Y âŠ† X. These are always true and contain no information.

3. Partial Dependency

A partial dependency occurs when a non-prime attribute (not part of any candidate key) is functionally determined by only part of a composite key. Partial dependencies violate 2NF.

Example: OrderDetails(order_id, product_id, quantity, product_name, unit_price) Composite key: {order_id, product_id} Full dependency: {order_id, product_id} â†’ quantity (quantity is fully dependent) Partial dep: product_id â†’ product_name (product_name depends on just product_id) Partial dep: product_id â†’ unit_price (unit_price depends on just product_id) These partial dependencies mean product_name is stored redundantly for every order line that includes that product â†’ update anomaly if the product is renamed.

4. Transitive Dependency

A transitive dependency occurs when a non-prime attribute depends on another non-prime attribute (indirectly through a chain). These violate 3NF.

A multivalued dependency (X â†  Y) is a stronger constraint: given X, Y can take multiple independent values that are not constrained by other attributes. These violate 4NF.

Armstrong's Axioms

Armstrong's axioms are a set of sound and complete inference rules for deriving all valid functional dependencies from a given set F. "Sound" means they only derive valid FDs; "complete" means they can derive all valid FDs.

Canonical Cover (Fc)

A canonical cover (minimal cover) Fc is a simplified, equivalent representation of a set of FDs where:

Dependency Preservation

A decomposition of relation R into R1, R2, ..., Rn is dependency-preserving if every FD in F+ can be verified using only the FDs within the individual decomposed relations (without needing to join them back together).
