Advanced SQL Joins, Subqueries, and Set Operations

SQL Advanced | 24 min read

Join Strategy

  • INNER JOIN for matching rows only.
  • LEFT JOIN for preserving left table rows.
  • FULL OUTER JOIN for symmetric inclusion.
  • SELF JOIN for hierarchical or comparative rows.

Subquery Types

  • Scalar subquery in SELECT/WHERE.
  • Table subquery in FROM (derived table).
  • Correlated subquery using outer query columns.

Set Operations

UNION removes duplicates, UNION ALL keeps duplicates, INTERSECT returns common rows, and EXCEPT returns rows in the first result not present in the second.

-- Employees earning above department average SELECT e.emp_id, e.name, e.salary FROM Employees e WHERE e.salary > ( SELECT AVG(e2.salary) FROM Employees e2 WHERE e2.dept_id = e.dept_id );
