SQL Essentials: DDL, DML, Operators & Functions

SQL Fundamentals | 22 min read

What Is SQL and Why Does It Dominate?

SQL (Structured Query Language) is the standard language for interacting with relational databases. Standardized in 1986 by ANSI and later ISO, it is supported (with minor dialect differences) by MySQL, PostgreSQL, SQL Server, Oracle, SQLite, and many others. Writing correct, efficient SQL is one of the most broadly useful skills in software engineering.

DDL: Defining Structure

CREATE TABLE

CREATE TABLE Students ( student_id INT PRIMARY KEY, name VARCHAR(100) NOT NULL, email VARCHAR(255) UNIQUE NOT NULL, dob DATE, gpa DECIMAL(3,2) CHECK (gpa >= 0.00 AND gpa <= 10.00), dept_id INT, enrolled_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP, FOREIGN KEY (dept_id) REFERENCES Departments(dept_id) ON DELETE SET NULL ON UPDATE CASCADE );

Key constraint types:

  • PRIMARY KEY — unique + not null.
  • UNIQUE — unique but allows one NULL per column (most DBMS).
  • NOT NULL — value required.
  • CHECK (expr) — value must satisfy expression.
  • DEFAULT value — used when no value is provided at insert time.
  • FOREIGN KEY ... REFERENCES — enforces referential integrity.

ALTER TABLE

-- Add a column ALTER TABLE Students ADD COLUMN phone VARCHAR(20); -- Modify a column type ALTER TABLE Students MODIFY COLUMN phone VARCHAR(15) NOT NULL; -- Rename a column (PostgreSQL syntax) ALTER TABLE Students RENAME COLUMN phone TO mobile; -- Drop a column ALTER TABLE Students DROP COLUMN mobile; -- Add a constraint ALTER TABLE Students ADD CONSTRAINT chk_gpa CHECK (gpa >= 0); -- Drop a constraint ALTER TABLE Students DROP CONSTRAINT chk_gpa;

DML: Manipulating Data

INSERT

-- Single row insert INSERT INTO Students (student_id, name, email, dob) VALUES (101, 'Mudit', 'mudit@example.com', '2002-04-15'); -- Multi-row insert INSERT INTO Students (student_id, name, email) VALUES (102, 'Aisha', 'aisha@example.com'), (103, 'Ravi', 'ravi@example.com'); -- Insert from a SELECT INSERT INTO ArchivedStudents (student_id, name, email) SELECT student_id, name, email FROM Students WHERE enrolled_at < '2020-01-01';

UPDATE

-- Update single column UPDATE Students SET gpa = 9.0 WHERE student_id = 101; -- Update multiple columns UPDATE Students SET gpa = 8.5, phone = '9876543210' WHERE student_id = 102; -- Update with calculation UPDATE Products SET price = price * 1.10 WHERE category = 'Electronics'; -- Update based on subquery UPDATE Students SET dept_id = (SELECT dept_id FROM Departments WHERE dept_name = 'Computer Science') WHERE student_id = 103;

DELETE

-- Delete specific rows DELETE FROM Enrollments WHERE student_id = 101 AND semester = '2023S1'; -- Delete all rows (but keep structure — use TRUNCATE for performance) DELETE FROM TemporaryLog; -- Delete with subquery DELETE FROM Students WHERE student_id NOT IN (SELECT DISTINCT student_id FROM Enrollments);

SELECT — The Core Query

SELECT [DISTINCT] column_list | * FROM table [alias] [JOIN ...] [WHERE condition] [GROUP BY columns] [HAVING group_condition] [ORDER BY columns [ASC | DESC]] [LIMIT n OFFSET m];

The logical processing order is: FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT.

Comparison and Logical Operators

-- Comparison WHERE age = 20 WHERE age != 20 OR WHERE age <> 20 WHERE gpa >= 8.5 WHERE gpa BETWEEN 7.0 AND 9.0 -- inclusive on both ends WHERE name LIKE 'M%' -- starts with M WHERE name LIKE '_u%' -- second char is u WHERE email ILIKE '%@gmail%' -- case-insensitive LIKE (PostgreSQL) WHERE dept_id IN (1, 2, 5) WHERE dept_id NOT IN (3, 4) WHERE dept_id IS NULL WHERE dept_id IS NOT NULL -- Logical WHERE age > 18 AND gpa >= 7.0 WHERE age > 25 OR gpa > 9.0 WHERE NOT (dept_id = 3) -- BETWEEN is equivalent to: WHERE gpa >= 7.0 AND gpa <= 9.0

SELECT dept_id, COUNT(*) AS total_students, AVG(gpa) AS avg_gpa, MAX(gpa) AS top_gpa, MIN(gpa) AS lowest_gpa, COUNT(DISTINCT dob) AS unique_birthdays FROM Students GROUP BY dept_id;

GROUP BY and HAVING

GROUP BY collapses rows with the same column values into one group. HAVING filters those groups (like WHERE, but applied after aggregation).

-- Find departments with more than 5 students and average GPA above 8 SELECT dept_id, COUNT(*) AS num_students, AVG(gpa) AS avg_gpa FROM Students GROUP BY dept_id HAVING COUNT(*) > 5 AND AVG(gpa) > 8.0 ORDER BY avg_gpa DESC;
You cannot use a column alias defined in SELECT inside HAVING. Repeat the aggregate expression: HAVING AVG(gpa) > 8.0, not HAVING avg_gpa > 8.0.

ORDER BY and LIMIT

-- Top 5 students by GPA descending SELECT student_id, name, gpa FROM Students ORDER BY gpa DESC, name ASC LIMIT 5; -- Pagination: page 3, 10 results per page (OFFSET = (page-1) * page_size) SELECT student_id, name FROM Students ORDER BY student_id LIMIT 10 OFFSET 20; -- skip first 20 rows, return next 10

NULL Handling in SQL

NULL in SQL means unknown or missing. It is not zero, it is not an empty string, and it is not false. Any arithmetic or comparison involving NULL returns NULL.

SELECT 5 + NULL; -- Returns NULL SELECT NULL = NULL; -- Returns NULL (not TRUE) SELECT NULL IS NULL; -- Returns TRUE SELECT NULL IS NOT NULL; -- Returns FALSE -- Use COALESCE to substitute a default value SELECT name, COALESCE(gpa, 0.0) AS gpa FROM Students; -- Use NULLIF to return NULL when two values are equal (avoid division by zero) SELECT total_sales / NULLIF(num_orders, 0) AS avg_order_value FROM SalesData;

String Functions

SELECT UPPER(name) -- 'mudit' → 'MUDIT' SELECT LOWER(name) -- 'MUDIT' → 'mudit' SELECT LENGTH(name) -- number of characters SELECT SUBSTRING(name, 1, 3) -- first 3 chars SELECT TRIM(' hello ') -- 'hello' SELECT LTRIM(name), RTRIM(name) -- trim left/right only SELECT CONCAT(first_name, ' ', last_name) -- join strings SELECT REPLACE(email, 'example', 'test') -- find and replace SELECT INSTR(email, '@') -- position of '@' SELECT LPAD(student_id, 6, '0') -- zero-pad: '000101'

Date and Time Functions

SELECT NOW() -- current date and time SELECT CURDATE() -- current date only (MySQL) SELECT CURRENT_DATE -- current date (standard SQL) SELECT DATEDIFF('2026-12-31', '2026-01-01') -- days between dates SELECT DATE_ADD(NOW(), INTERVAL 30 DAY) -- 30 days from now SELECT YEAR(dob), MONTH(dob), DAY(dob) FROM Students SELECT EXTRACT(YEAR FROM enrolled_at) AS yr -- standard SQL SELECT TO_CHAR(now(), 'YYYY-MM-DD') -- PostgreSQL format

CASE Expressions

CASE is SQL's conditional logic — equivalent to if/else inside a query.

-- Simple CASE SELECT name, CASE grade WHEN 'A+' THEN 'Outstanding' WHEN 'A' THEN 'Excellent' WHEN 'B' THEN 'Good' ELSE 'Needs Improvement' END AS performance FROM Enrollments; -- Searched CASE (more powerful) SELECT name, gpa, CASE WHEN gpa >= 9.0 THEN 'Distinction' WHEN gpa >= 7.5 THEN 'First Class' WHEN gpa >= 6.0 THEN 'Second Class' ELSE 'Pass' END AS grade_category FROM Students;

Subqueries in SELECT

-- Scalar subquery: returns single value SELECT name, gpa, (SELECT AVG(gpa) FROM Students) AS overall_avg FROM Students; -- IN with subquery SELECT name FROM Students WHERE dept_id IN ( SELECT dept_id FROM Departments WHERE dept_name LIKE '%Science%' ); -- EXISTS (often more efficient than IN for large result sets) SELECT name FROM Students s WHERE EXISTS ( SELECT 1 FROM Enrollments e WHERE e.student_id = s.student_id ); -- Correlated subquery (references outer query) SELECT name, gpa FROM Students s WHERE gpa > ( SELECT AVG(gpa) FROM Students WHERE dept_id = s.dept_id );

TCL: Transaction Control

BEGIN; -- or START TRANSACTION UPDATE Accounts SET balance = balance - 500 WHERE account_id = 1; UPDATE Accounts SET balance = balance + 500 WHERE account_id = 2; -- If both succeed: COMMIT; -- If anything goes wrong: ROLLBACK; -- Partial rollback with savepoint SAVEPOINT before_transfer; UPDATE Accounts SET balance = balance - 500 WHERE account_id = 1; -- Something went wrong ROLLBACK TO before_transfer; COMMIT;

DCL: GRANT and REVOKE

-- Grant SELECT privilege on Students table to user 'analyst' GRANT SELECT ON Students TO analyst; -- Grant multiple privileges GRANT SELECT, INSERT, UPDATE ON Enrollments TO app_user; -- Grant all privileges GRANT ALL PRIVILEGES ON DATABASE university TO admin_user; -- Revoke a privilege REVOKE INSERT ON Students FROM analyst; -- Grant with pass-on ability GRANT SELECT ON Students TO manager WITH GRANT OPTION;

Common Pitfalls

  • Selecting non-aggregated columns without GROUP BY: Causes errors in strict mode (MySQL 5.7+, PostgreSQL always). Every column in SELECT must either be in GROUP BY or wrapped in an aggregate.
  • Confusing WHERE and HAVING: WHERE filters rows before aggregation; HAVING filters groups after aggregation.
  • NULL comparisons: Use IS NULL / IS NOT NULL, never = NULL.
  • Missing quotes on string literals: Always use single quotes for strings in SQL, never double quotes (double quotes mean identifiers in standard SQL).
  • TRUNCATE is hard to rollback: Prefer DELETE inside a transaction when you need rollback safety.

What You Should Remember

  • DDL defines structure (CREATE, ALTER, DROP). Changes are usually auto-committed.
  • DML manipulates data (SELECT, INSERT, UPDATE, DELETE). Participates in transactions.
  • SELECT logical order: FROM → JOIN → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT.
  • Aggregates ignore NULLs except COUNT(*). Handle NULLs deliberately with COALESCE, NULLIF, IS NULL.
  • CASE expressions let you put conditional logic directly in a query without application code.
  • Subqueries can appear in SELECT, FROM, WHERE, and HAVING clauses.
