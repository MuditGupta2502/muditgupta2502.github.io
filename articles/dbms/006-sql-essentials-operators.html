SQL Essentials: DDL, DML, Operators & Fundamentals

SQL Fundamentals | 10 min read

What is SQL?

SQL (Structured Query Language) is a standardized programming language used to manage and manipulate relational databases. It provides a consistent interface across different DBMS platforms (MySQL, PostgreSQL, Oracle, SQL Server).

SQL has three main categories:

  • DDL: Data Definition Language (CREATE, ALTER, DROP)
  • DML: Data Manipulation Language (SELECT, INSERT, UPDATE, DELETE)
  • DCL: Data Control Language (GRANT, REVOKE)

DDL: Data Definition Language

DDL statements define the structure of the database and tables.

CREATE TABLE: Creates a new table with specified columns and constraints. 

CREATE TABLE Students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  name VARCHAR(100) NOT NULL,
  email VARCHAR(100) UNIQUE,
  age INT CHECK (age >= 18),
  enrollment_date DATE DEFAULT CURDATE()
);

ALTER TABLE: Modifies an existing table structure. 

-- Add a column
ALTER TABLE Students ADD phone VARCHAR(15);

-- Drop a column
ALTER TABLE Students DROP COLUMN phone;

-- Modify column
ALTER TABLE Students MODIFY name VARCHAR(150);

DROP TABLE: Deletes a table and all its data. 

DROP TABLE Students;
DROP TABLE IF EXISTS Students; -- Safe drop

TRUNCATE TABLE: Deletes all rows but keeps the table structure. 

TRUNCATE TABLE Students;

DML: Data Manipulation Language

SELECT: Retrieves data from tables. 

SELECT column1, column2 FROM table_name;
SELECT * FROM Students; -- All columns
SELECT name, email FROM Students WHERE age > 20;

INSERT: Adds new rows to a table. 

INSERT INTO Students (name, email, age)
VALUES ('John Doe', 'john@example.com', 22);

INSERT INTO Students 
VALUES (1, 'Jane Doe', 'jane@example.com', 21, '2024-01-15');

UPDATE: Modifies existing data. 

UPDATE Students 
SET email = 'newemail@example.com' 
WHERE student_id = 1;

UPDATE Students 
SET age = age + 1; -- Increment all ages

DELETE: Removes rows from a table. 

DELETE FROM Students WHERE student_id = 1;
DELETE FROM Students WHERE age < 18;

Comparison Operators

  • = Equal to: WHERE age = 25
  • != or <> Not equal: WHERE status != 'inactive'
  • > Greater than: WHERE salary > 50000
  • < Less than: WHERE age < 30
  • >= Greater or equal: WHERE score >= 80
  • <= Less or equal: WHERE quantity <= 10

Logical Operators

AND: All conditions must be true. 

SELECT * FROM Students 
WHERE age > 20 AND grade = 'A';

OR: At least one condition must be true. 

SELECT * FROM Students 
WHERE status = 'active' OR age > 25;

NOT: Negates a condition. 

SELECT * FROM Students 
WHERE NOT status = 'inactive';

Pattern Matching: LIKE

LIKE is used for pattern matching with wildcards. 

-- Start with 'J'
SELECT * FROM Students WHERE name LIKE 'J%';

-- End with 'son'
SELECT * FROM Students WHERE name LIKE '%son';

-- Contains 'oh'
SELECT * FROM Students WHERE name LIKE '%oh%';

-- Exact 4 characters
SELECT * FROM Students WHERE name LIKE '____';

IN and BETWEEN

IN: Matches values in a list. 

SELECT * FROM Students 
WHERE student_id IN (1, 5, 10);

BETWEEN: Matches range of values (inclusive). 

SELECT * FROM Students 
WHERE age BETWEEN 20 AND 25;

NULL Checks

-- Find NULL values
SELECT * FROM Students WHERE email IS NULL;

-- Find non-NULL values
SELECT * FROM Students WHERE email IS NOT NULL;

Aggregate Functions

  • COUNT(): Counts number of rows
  • SUM(): Sums numeric values
  • AVG(): Calculates average
  • MAX(): Finds maximum value
  • MIN(): Finds minimum value
SELECT COUNT(*) FROM Students;
SELECT AVG(age) FROM Students;
SELECT MAX(salary) FROM Employees;

ORDER BY and GROUP BY

ORDER BY: Sorts result set. 

SELECT * FROM Students 
ORDER BY age DESC;

SELECT * FROM Students 
ORDER BY name ASC, age DESC;

GROUP BY: Groups rows with aggregate functions. 

SELECT department, COUNT(*) as emp_count
FROM Employees
GROUP BY department;

Constraints

  • PRIMARY KEY: Uniquely identifies each row
  • FOREIGN KEY: References primary key of another table
  • UNIQUE: All values in column are different
  • NOT NULL: Column must have a value
  • CHECK: Ensures values satisfy a condition
  • DEFAULT: Provides default value if none specified

Summary

SQL is the fundamental language for interacting with relational databases. Understanding DDL for defining structures, DML for manipulating data, and various operators for filtering and transforming data is essential for any database professional. These basics form the foundation for more advanced SQL topics like joins, subqueries, and optimization.
