Advanced SQL: Joins, Subqueries & Complex Queries

SQL Advanced | 11 min read

SQL Joins: Combining Data from Multiple Tables

Joins are used to combine rows from two or more tables based on related columns. They are fundamental for querying normalized databases.

Inner Join

Returns rows that have matching values in both tables. Only records with relationships are included. 

SELECT Students.name, Courses.course_name
FROM Students
INNER JOIN Enrollments ON Students.student_id = Enrollments.student_id
INNER JOIN Courses ON Enrollments.course_id = Courses.course_id;

Left Outer Join

Returns all rows from the left table and matching rows from the right table. Non-matching right rows show NULL. 

-- All students and their enrollments (even if not enrolled)
SELECT Students.name, Courses.course_name
FROM Students
LEFT JOIN Enrollments ON Students.student_id = Enrollments.student_id
LEFT JOIN Courses ON Enrollments.course_id = Courses.course_id;

Right Outer Join

Returns all rows from the right table and matching rows from the left table. Non-matching left rows show NULL. 

-- All courses and their enrolled students (even if no students enrolled)
SELECT Students.name, Courses.course_name
FROM Students
RIGHT JOIN Enrollments ON Students.student_id = Enrollments.student_id
RIGHT JOIN Courses ON Enrollments.course_id = Courses.course_id;

Full Outer Join

Returns all rows from both tables. Non-matching rows show NULL on both sides. (Note: Not all DBMS support FULL JOIN; use UNION instead) 

-- MySQL alternative using UNION
(SELECT Students.name, Courses.course_name
 FROM Students
 LEFT JOIN Enrollments ON Students.student_id = Enrollments.student_id
 LEFT JOIN Courses ON Enrollments.course_id = Courses.course_id)
UNION
(SELECT Students.name, Courses.course_name
 FROM Students
 RIGHT JOIN Enrollments ON Students.student_id = Enrollments.student_id
 RIGHT JOIN Courses ON Enrollments.course_id = Courses.course_id);

Cross Join

Produces Cartesian product — all possible combinations of rows from both tables. 

-- All possible combinations of students and courses
SELECT Students.name, Courses.course_name
FROM Students
CROSS JOIN Courses;

Self Join

Joins a table to itself. Useful for hierarchical data like employee-manager relationships. 

SELECT e1.name as Employee, e2.name as Manager
FROM Employees e1
INNER JOIN Employees e2 ON e1.manager_id = e2.employee_id;

Subqueries

A subquery (inner query) is a query within another query. Used to filter or aggregate data.

Subquery in WHERE Clause

-- Find students with above average age
SELECT name, age
FROM Students
WHERE age > (SELECT AVG(age) FROM Students);

Subquery with IN

-- Find students enrolled in specific courses
SELECT name
FROM Students
WHERE student_id IN (
  SELECT student_id FROM Enrollments 
  WHERE course_id IN (1, 2, 3)
);

Correlated Subquery

A subquery that references columns from the outer query. Executed once for each row. 

-- Find students with above department average salary
SELECT name, salary
FROM Employees e1
WHERE salary > (
  SELECT AVG(salary) 
  FROM Employees e2
  WHERE e1.department_id = e2.department_id
);

Subquery in FROM Clause (Derived Table)

-- Average age by department
SELECT dept.department, dept.avg_age
FROM (
  SELECT department_id, AVG(age) as avg_age
  FROM Employees
  GROUP BY department_id
) dept;

Views

A view is a virtual table created from a query result. It doesn't store data but provides a logical view of existing tables.

Creating a View: 

CREATE VIEW StudentEnrollment AS
SELECT Students.name, Courses.course_name, Enrollments.enrollment_date
FROM Students
INNER JOIN Enrollments ON Students.student_id = Enrollments.student_id
INNER JOIN Courses ON Enrollments.course_id = Courses.course_id;

Using a View: 

SELECT * FROM StudentEnrollment WHERE name = 'John Doe';

Dropping a View: 

DROP VIEW StudentEnrollment;

Common Table Expressions (CTE)

CTE is a named temporary result set defined using WITH clause. Makes complex queries more readable. 

WITH HighEarners AS (
  SELECT employee_id, name, salary
  FROM Employees
  WHERE salary > 80000
)
SELECT * FROM HighEarners
WHERE department_id = 2;

Window Functions

Window functions perform calculations across rows related to the current row. 

-- Rank employees by salary within department
SELECT name, salary, department_id,
       RANK() OVER (PARTITION BY department_id ORDER BY salary DESC) as rank
FROM Employees;

UNION vs UNION ALL

UNION: Combines results from multiple queries and removes duplicates. 

SELECT name FROM Students
UNION
SELECT name FROM Faculty;

UNION ALL: Combines results and keeps duplicates (faster). 

SELECT name FROM Students
UNION ALL
SELECT name FROM Faculty;

CASE Statement

Provides conditional logic within queries. 

SELECT name, age,
  CASE
    WHEN age >= 21 THEN 'Adult'
    WHEN age >= 13 THEN 'Teenager'
    ELSE 'Child'
  END as age_group
FROM Students;

Summary

Advanced SQL techniques like joins, subqueries, views, and CTEs enable complex data retrieval from normalized databases. Mastering these concepts is essential for writing efficient, maintainable queries that handle real-world data scenarios.
