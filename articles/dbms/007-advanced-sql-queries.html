Advanced SQL: Joins, Subqueries, Window Functions & CTEs

SQL Advanced | 25 min read

Sample Schema Used in Examples

Students (student_id, name, dept_id, gpa) Departments (dept_id, dept_name) Courses (course_id, course_name, credits) Enrollments (student_id, course_id, semester, grade) Employees (emp_id, name, salary, dept_id, manager_id)

All JOIN Types

A JOIN combines rows from two or more tables according to a relationship expression. The type of JOIN controls what happens when no match exists.

INNER JOIN

Returns only rows where the ON condition is satisfied in both tables. Non-matching rows from both sides are excluded.

SELECT s.name, d.dept_name FROM Students s INNER JOIN Departments d ON s.dept_id = d.dept_id; -- Only students who have a matching department are returned. -- Students with NULL dept_id, or departments with no students, are excluded.

LEFT (OUTER) JOIN

Returns all rows from the left table plus matching rows from the right table. Where there is no match, right-side columns are NULL.

SELECT s.name, d.dept_name FROM Students s LEFT JOIN Departments d ON s.dept_id = d.dept_id; -- All students appear; dept_name is NULL for students not assigned to any department.

RIGHT (OUTER) JOIN

Mirror of LEFT JOIN. Returns all rows from the right table; unmatched left-side columns are NULL.

SELECT s.name, d.dept_name FROM Students s RIGHT JOIN Departments d ON s.dept_id = d.dept_id; -- All departments appear; name is NULL for departments with no students.

FULL (OUTER) JOIN

Returns all rows from both tables. Unmatched rows from either side get NULLs for the other side's columns.

SELECT s.name, d.dept_name FROM Students s FULL OUTER JOIN Departments d ON s.dept_id = d.dept_id; -- All students + all departments. NULLs where no match exists on either side.

CROSS JOIN

Cartesian product — every row from the left combined with every row from the right. No ON clause.

SELECT s.name, c.course_name FROM Students s CROSS JOIN Courses c; -- If 100 students and 50 courses: 5,000 rows. Use intentionally.

SELF JOIN

A table joined to itself. Requires aliases to distinguish the two "copies."

-- Find each employee's manager name SELECT e.name AS employee, m.name AS manager FROM Employees e LEFT JOIN Employees m ON e.manager_id = m.emp_id; -- e represents the employee; m represents the manager (same table)

Finding Unmatched Rows (Anti-JOIN pattern)

-- Students with no enrollments SELECT s.name FROM Students s LEFT JOIN Enrollments e ON s.student_id = e.student_id WHERE e.student_id IS NULL; -- Equivalent using NOT EXISTS SELECT name FROM Students s WHERE NOT EXISTS ( SELECT 1 FROM Enrollments e WHERE e.student_id = s.student_id );

JOIN on Multiple Conditions

SELECT s.name, e.course_id, e.grade FROM Students s JOIN Enrollments e ON s.student_id = e.student_id AND e.semester = '2026S1' WHERE s.gpa >= 8.0;

Multi-Table JOIN

SELECT s.name, d.dept_name, c.course_name, e.grade FROM Students s JOIN Departments d ON s.dept_id = d.dept_id JOIN Enrollments e ON s.student_id = e.student_id JOIN Courses c ON e.course_id = c.course_id WHERE e.semester = '2026S1' ORDER BY s.name;

Subqueries — In Depth

Scalar Subquery

Returns exactly one row and one column. Can be used anywhere a single value is expected.

-- Compare each student's GPA to the overall average SELECT name, gpa, gpa - (SELECT AVG(gpa) FROM Students) AS diff_from_avg FROM Students ORDER BY diff_from_avg DESC;

Row Subquery

-- Find students whose (name, dept_id) matches a specific combination SELECT * FROM Students WHERE (name, dept_id) = (SELECT name, dept_id FROM Students WHERE student_id = 101) AND student_id != 101;

Table Subquery (Derived Table / Inline View)

-- Average GPA per department using a subquery in FROM SELECT dept_id, ROUND(avg_gpa, 2) FROM ( SELECT dept_id, AVG(gpa) AS avg_gpa FROM Students GROUP BY dept_id ) AS dept_averages WHERE avg_gpa >= 7.5;

Correlated Subquery

References the outer query's current row. Evaluated once per row in the outer query — can be slow on large tables but very expressive.

-- Students with GPA above their department average SELECT name, gpa, dept_id FROM Students s WHERE gpa > ( SELECT AVG(gpa) FROM Students WHERE dept_id = s.dept_id -- references outer query ); -- N-th highest salary per department (correlated) SELECT name, salary, dept_id FROM Employees e1 WHERE salary = ( SELECT MAX(salary) FROM Employees e2 WHERE e2.dept_id = e1.dept_id -- correlated: same department only );

EXISTS and NOT EXISTS

EXISTS returns TRUE if the subquery returns at least one row. It short-circuits: stops scanning as soon as one row is found.

-- Students who have enrolled in at least one course SELECT name FROM Students s WHERE EXISTS ( SELECT 1 FROM Enrollments e WHERE e.student_id = s.student_id ); -- Students who have never enrolled SELECT name FROM Students s WHERE NOT EXISTS ( SELECT 1 FROM Enrollments e WHERE e.student_id = s.student_id );

Common Table Expressions (CTEs)

A CTE (WITH clause) defines a named temporary result set that can be referenced in the main query. CTEs improve readability and allow step-by-step query construction.

Basic CTE

WITH TopStudents AS ( SELECT student_id, name, gpa FROM Students WHERE gpa >= 9.0 ) SELECT ts.name, ts.gpa, d.dept_name FROM TopStudents ts JOIN Departments d ON ts.dept_id = d.dept_id;

Multiple CTEs

WITH DeptAverages AS ( SELECT dept_id, AVG(gpa) AS avg_gpa FROM Students GROUP BY dept_id ), EnrolledStudents AS ( SELECT DISTINCT student_id FROM Enrollments WHERE semester = '2026S1' ) SELECT s.name, s.gpa, da.avg_gpa FROM Students s JOIN DeptAverages da ON s.dept_id = da.dept_id JOIN EnrolledStudents es ON s.student_id = es.student_id WHERE s.gpa > da.avg_gpa;

Recursive CTE

Recursive CTEs allow traversal of hierarchical data like org charts, file systems, or category trees.

-- Find all reports under employee 1 (org chart traversal) WITH RECURSIVE OrgTree AS ( -- Base case: the root employee SELECT emp_id, name, manager_id, 0 AS depth FROM Employees WHERE emp_id = 1 UNION ALL -- Recursive step: join employees to their parent in OrgTree SELECT e.emp_id, e.name, e.manager_id, ot.depth + 1 FROM Employees e JOIN OrgTree ot ON e.manager_id = ot.emp_id ) SELECT emp_id, name, depth FROM OrgTree ORDER BY depth, name;

Window Functions

Window functions perform calculations across a set of rows related to the current row, without collapsing rows the way GROUP BY does. They are evaluated after WHERE, GROUP BY, and HAVING.

function_name(args) OVER ( [PARTITION BY partition_cols] [ORDER BY order_cols] [ROWS|RANGE BETWEEN frame_start AND frame_end] )

ROW_NUMBER, RANK, DENSE_RANK, NTILE

SELECT name, dept_id, gpa, ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY gpa DESC) AS row_num, RANK() OVER (PARTITION BY dept_id ORDER BY gpa DESC) AS rnk, DENSE_RANK() OVER (PARTITION BY dept_id ORDER BY gpa DESC) AS dense_rnk, NTILE(4) OVER (PARTITION BY dept_id ORDER BY gpa DESC) AS quartile FROM Students; -- ROW_NUMBER: unique sequential number (1, 2, 3, 4...) -- RANK: skips numbers for ties (1, 1, 3, 4...) -- DENSE_RANK: no skipping for ties (1, 1, 2, 3...) -- NTILE(n): divides rows into n buckets

Get Top-N Per Group (using ROW_NUMBER)

-- Top 2 students per department SELECT name, dept_id, gpa FROM ( SELECT name, dept_id, gpa, ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY gpa DESC) AS rn FROM Students ) ranked WHERE rn <= 2;

LAG and LEAD

-- Compare each month's sales to previous month SELECT month, sales, LAG(sales, 1) OVER (ORDER BY month) AS prev_month, LEAD(sales, 1) OVER (ORDER BY month) AS next_month, sales - LAG(sales, 1) OVER (ORDER BY month) AS month_over_month FROM MonthlySales;

Running Totals and Moving Averages (Frames)

SELECT month, sales, -- Running total from start up to current row SUM(sales) OVER (ORDER BY month ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW) AS running_total, -- 3-month moving average (current + 2 preceding) AVG(sales) OVER (ORDER BY month ROWS BETWEEN 2 PRECEDING AND CURRENT ROW) AS moving_avg_3m FROM MonthlySales;

FIRST_VALUE and LAST_VALUE

SELECT name, dept_id, gpa, FIRST_VALUE(name) OVER (PARTITION BY dept_id ORDER BY gpa DESC) AS top_student, LAST_VALUE(name) OVER ( PARTITION BY dept_id ORDER BY gpa DESC ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING ) AS bottom_student FROM Students;

ROLLUP and CUBE (Multidimensional Aggregation)

ROLLUP

Generates subtotals and a grand total for a hierarchy of columns.

SELECT dept_id, semester, COUNT(*) AS enrollments FROM Enrollments GROUP BY ROLLUP(dept_id, semester); -- Produces: -- Rows grouped by (dept_id, semester) -- Subtotals grouped by (dept_id) only -- Grand total row (NULL, NULL)

CUBE

Generates aggregates for all possible combinations of the listed columns.

SELECT dept_id, semester, COUNT(*) FROM Enrollments e JOIN Students s ON e.student_id = s.student_id GROUP BY CUBE(dept_id, semester); -- All combinations: (dept_id, semester), (dept_id), (semester), ()

INTERSECT and EXCEPT (Set Operations)

-- Students enrolled in BOTH DBMS (C001) AND OS (C002) SELECT student_id FROM Enrollments WHERE course_id = 'C001' INTERSECT SELECT student_id FROM Enrollments WHERE course_id = 'C002'; -- Students enrolled in DBMS but NOT in OS SELECT student_id FROM Enrollments WHERE course_id = 'C001' EXCEPT SELECT student_id FROM Enrollments WHERE course_id = 'C002'; -- PostgreSQL uses EXCEPT; MySQL uses MINUS

PIVOT Queries (Conditional Aggregation)

SQL doesn't always have a PIVOT keyword, but conditional aggregation with CASE achieves the same result portably.

-- Count enrollments per semester, pivoted as columns SELECT dept_id, COUNT(CASE WHEN semester = '2025S1' THEN 1 END) AS s2025_1, COUNT(CASE WHEN semester = '2025S2' THEN 1 END) AS s2025_2, COUNT(CASE WHEN semester = '2026S1' THEN 1 END) AS s2026_1 FROM Enrollments e JOIN Students s ON e.student_id = s.student_id GROUP BY dept_id;

Finding Duplicates

-- Find emails that appear more than once SELECT email, COUNT(*) AS occurrences FROM Students GROUP BY email HAVING COUNT(*) > 1; -- Delete duplicates keeping the lowest student_id DELETE FROM Students WHERE student_id NOT IN ( SELECT MIN(student_id) FROM Students GROUP BY email );

Practical Interview Patterns

-- Second highest salary SELECT MAX(salary) AS second_highest FROM Employees WHERE salary < (SELECT MAX(salary) FROM Employees); -- Or using DENSE_RANK SELECT salary FROM ( SELECT salary, DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk FROM Employees ) r WHERE rnk = 2; -- Employees earning more than their department average SELECT e.name, e.salary, e.dept_id FROM Employees e JOIN ( SELECT dept_id, AVG(salary) AS avg_sal FROM Employees GROUP BY dept_id ) d ON e.dept_id = d.dept_id WHERE e.salary > d.avg_sal;

What You Should Remember

  • INNER JOIN returns only matched rows. LEFT/RIGHT/FULL OUTER JOINs preserve unmatched rows from one or both sides.
  • SELF JOIN is used for hierarchical or comparative queries on the same table — always alias both sides.
  • Anti-JOIN pattern: LEFT JOIN + WHERE right.key IS NULL, or NOT EXISTS.
  • CTEs (WITH clause) name intermediate results, making complex queries modular and readable. Recursive CTEs handle tree traversal.
  • Window functions compute over a window without collapsing rows. PARTITION BY resets the window per group; ORDER BY defines the sequence within the window.
  • ROW_NUMBER gives unique ranks; RANK skips after ties; DENSE_RANK does not skip.
  • ROLLUP generates a subtotal hierarchy; CUBE generates all combinations.
  • Conditional aggregation with CASE inside SUM/COUNT is the portable PIVOT technique.
