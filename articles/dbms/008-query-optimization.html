Query Optimization: Making Queries Fast & Efficient

Performance & Optimization | 10 min read

What is Query Optimization?

Query optimization is the process of improving SQL query performance by minimizing resource consumption (CPU, memory, disk I/O) and execution time. The DBMS query optimizer automatically evaluates different execution plans and chooses the most efficient one.

Goal: Retrieve required data with minimum computational resources.

Query Processing Steps

  1. Parsing: Check SQL syntax validity
  2. Translation: Convert SQL to internal representation
  3. Optimization: Generate optimal execution plan
  4. Compilation: Convert to executable code
  5. Execution: Execute the compiled plan

Understanding Execution Plans

An execution plan shows how the DBMS executes a query. It reveals the operations, join methods, and order of data access.

View Execution Plan: 

-- MySQL
EXPLAIN SELECT * FROM Students WHERE age > 20;
EXPLAIN FORMAT=JSON SELECT ...;

-- PostgreSQL
EXPLAIN SELECT * FROM Students WHERE age > 20;
EXPLAIN ANALYZE SELECT *;

Indexing Strategy

An index is a data structure that enables fast data retrieval, similar to a book's index. Without indexes, the DBMS performs full table scans.

Types of Indexes

Single-Column Index: Index on a single column. 

CREATE INDEX idx_name ON Students(name);
CREATE INDEX idx_email ON Students(email);

Composite (Multi-Column) Index: Index on multiple columns. 

CREATE INDEX idx_dept_salary ON Employees(department_id, salary);

Unique Index: Enforces uniqueness. 

CREATE UNIQUE INDEX idx_email ON Students(email);

Full-Text Index: For text search. 

CREATE FULLTEXT INDEX idx_content ON Articles(content);

When to Create Indexes

  • WHERE Clause: Frequently filtered columns
  • JOIN Conditions: Columns used in JOIN ON
  • ORDER BY/GROUP BY: Sorted or grouped columns
  • Foreign Keys: Improve JOIN performance

When NOT to Index

  • Small tables (less than 1000 rows)
  • Columns with low selectivity (few unique values)
  • Columns updated frequently (maintenance overhead)
  • Columns rarely used in WHERE/JOIN

Query Optimization Techniques

1. Use Selective Columns

Avoid: 

SELECT * FROM Students;

Prefer: 

SELECT student_id, name, email FROM Students;

2. Add WHERE Clause Conditions

Reduce rows early to minimize processing. 

-- Better: Filters early
SELECT * FROM Students 
WHERE department_id = 5 AND age > 20;

-- Worse: No filtering
SELECT * FROM Students

3. Use INNER JOIN Instead of Outer Joins

INNER joins are faster because they process fewer rows. 

-- Prefer INNER when you need matching rows only
SELECT s.name, c.course_name
FROM Students s
INNER JOIN Enrollments e ON s.student_id = e.student_id
INNER JOIN Courses c ON e.course_id = c.course_id;

4. Avoid Functions in WHERE Clause

Avoid: Functions prevent using indexes. 

SELECT * FROM Students 
WHERE UPPER(name) = 'JOHN'; -- Can't use index

Prefer: 

SELECT * FROM Students 
WHERE name = 'John'; -- Can use index

5. Use EXISTS Instead of IN for Subqueries

Better Performance: 

SELECT * FROM Students s
WHERE EXISTS (
  SELECT 1 FROM Enrollments 
  WHERE student_id = s.student_id
);

6. Parameterize Queries

Use prepared statements to prevent SQL injection and enable query caching. 

-- Prepared statement (pseudo-code)
PREPARE stmt FROM 'SELECT * FROM Students WHERE student_id = ?'
EXECUTE stmt USING 5;

7. Limit Result Set

SELECT * FROM Students LIMIT 10;

8. Use Denormalization for Read-Heavy Queries

Intentionally store redundant data for frequently accessed aggregates. 

-- Add order_total to Orders table even though it can be calculated
ALTER TABLE Orders ADD order_total DECIMAL(10,2);

Batch Operations

Insert/update multiple rows at once instead of individual statements. 

-- Better: Batch insert
INSERT INTO Students (name, email) VALUES
('John', 'john@ex.com'),
('Jane', 'jane@ex.com'),
('Bob', 'bob@ex.com');

-- Avoid: Individual inserts in a loop

Database Statistics

Database maintainers should regularly update statistics so the optimizer makes good decisions. 

-- MySQL
ANALYZE TABLE Students;

-- PostgreSQL
ANALYZE;

Summary

Query optimization is a critical skill for database performance. By understanding execution plans, using appropriate indexes, writing selective queries, and avoiding common pitfalls, you can dramatically improve application responsiveness. Regular monitoring and tuning ensure databases remain performant as data grows.
