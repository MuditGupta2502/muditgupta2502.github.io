Functional Dependencies, Armstrong Axioms, Closure, and Denormalization

Normalization | 21 min read

Functional Dependency Refresher

For relation R, X -> Y means each X value determines exactly one Y value. FDs define redundancy and normal form violations.

Armstrong Axioms

  • Reflexivity: if Y subset of X then X -> Y.
  • Augmentation: if X -> Y then XZ -> YZ.
  • Transitivity: if X -> Y and Y -> Z then X -> Z.

Derived rules (union, decomposition, pseudo-transitivity) make reasoning practical in exam and interview settings.

Attribute Closure

Given F = {A->B, B->C, CD->E} A+ = {A, B, C} CD+ = {C, D, E}

Compute closure to test superkeys, candidate keys, and dependency preservation after decomposition.

Inference Rules and Minimal Cover

  • Split right sides into single attributes.
  • Remove extraneous attributes from left sides.
  • Drop redundant dependencies.

Denormalization Trade-off

Denormalization improves read performance at the cost of update complexity and potential inconsistency. Apply only when measured bottlenecks justify it.

Use normalization for correctness first, denormalize selectively for measured performance gains.
