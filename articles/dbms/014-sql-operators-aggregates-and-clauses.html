SQL Operators, Aggregates, and Clause Evaluation Order

SQL Core | 22 min read

Core SQL Operators

  • Comparison: =, !=, >, <, >=, <=
  • Logical: AND, OR, NOT
  • Set membership: IN, EXISTS
  • Pattern matching: LIKE, ILIKE

Aggregate Functions

FunctionPurpose
COUNTCount rows or non-null values
SUMTotal numeric values
AVGMean value
MIN / MAXExtremes

Clause Evaluation Order

Logical order: FROM -> JOIN -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY -> LIMIT

Practical Example

SELECT dept_id, COUNT(*) AS staff_count, AVG(salary) AS avg_salary FROM Employees WHERE active = TRUE GROUP BY dept_id HAVING COUNT(*) >= 5 ORDER BY avg_salary DESC;

Understanding logical order explains why aliases from SELECT are usually unavailable in WHERE and why HAVING exists.
